推送到远程库

git fetch是将远程主机的最新内容拉到本地，用户在检查了以后决定是否合并到工作本机分支中。

git pull 则是将远程主机的最新内容拉下来后直接合并，即：git pull = git fetch + git merge，这样可能会产生冲突，需要手动解决。

推送冲突

  1. pull
  2. 解决冲突，编辑文档
  3. 添加暂存区 git add [文件]
  4. 提交到本地库 git commit -m "----"
  5. 推送到远程库 git push [远程仓库别名] [分支]
